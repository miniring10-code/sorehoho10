# Claude Code 指示書
## それからふくらむ可愛い頬を、10周年公演 管理アプリ

---

## 概要

アイドルコピーダンスサークルの11月公演に向けた、メンバー間で共有できる管理Webアプリを作ってほしい。
**各自がスマホからアクセスして、入力した内容がリアルタイムで全員に反映される**ことが最重要条件。

---

## 必須要件

- スマホで快適に使えること（モバイルファースト）
- メンバー全員がURLを開くだけで使えること（ログイン不要が望ましい）
- 誰かが入力したデータが全員の画面にリアルタイムで反映されること
- コードがわからない人でも運用できること

---

## 機能一覧

### 1. スケジュール・カレンダー
- 月表示のカレンダー
- イベントのある日にドット表示
- 月の移動（前月・次月）
- 今後の予定一覧（イベント名・日付）
- イベントの追加・編集・削除

### 2. タスク管理
- カテゴリ別フィルター（横スクロールのチップUI）
- カテゴリ一覧：sns・会場・衣装・クリエイティブ・現役連絡・ロゴ・kv・演出・音源・チケット・グッズ・お金・スタジオ予約・スケジュール
- タスクのタップで完了/未完了の切り替え
- 完了済みタスクは下部に分離して表示
- タスクの追加（名前・カテゴリを入力）

### 3. お知らせ・掲示板
- 投稿一覧（タイトル・カテゴリバッジ・投稿時間・NEWバッジ）
- タイトルをタップすると詳細がアコーディオン展開
- 投稿カテゴリ：全体・衣装・練習・会場・グッズ
- 新規投稿モーダル（タイトル・詳細文・カテゴリ選択）

### 4. 出欠確認
- イベントごとに出欠を管理
- メンバーごとに「○（参加）・△（未定）・✕（不参加）」を選択
- 参加/不参加/未定の集計数をリアルタイム表示
- 地方メンバー向けの「上京メモ」欄（自由記述）
- イベントの追加・編集・削除

### 5. セトリ（セットリスト）
- 曲の一覧表示（番号・曲名・アーティスト名）
- 曲をタップすると詳細ページへ遷移（右スライドアニメーション）
- 詳細ページ内容：
  - パート分け・フォーメーション（メンバーごとに役割を入力・保存）
  - 連絡事項（テキスト形式で追加・削除可能）

### 6. 設定ページ
- **メンバー管理**：名前の編集・追加・削除・地方フラグの切り替え
- **セトリ管理**：曲名・アーティスト名の編集・追加・削除・順番の入れ替え（▲▼）
- **出欠イベント管理**：イベント名・日時の編集・追加・削除
- 「変更を適用する」ボタンで全体に反映

## 技術スタック（推奨）

リアルタイム共有を実現するため、以下を推奨：

### フロントエンド
- HTML / CSS / JavaScript（またはReact）
- デザインは上記仕様に準拠

### バックエンド・データ同期
以下のいずれかを使用してリアルタイム同期を実現すること：

**オプションA（最もかんたん）**
- Firebase Realtime Database または Firestore（無料枠あり）
- 認証不要でURLだけで全員がアクセスできる構成

**オプションB**
- Supabase（PostgreSQL + リアルタイム機能）

**オプションC**
- PocketBase（セルフホスト）

### ホスティング
- Firebase Hosting（無料）
- または Vercel / Netlify（無料）

---

## 運用要件

- メンバーはURLをブラウザで開くだけで使える
- アプリのインストール不要
- ログイン・アカウント登録不要（もしくは最小限）
- スマホのホーム画面に追加できるとなお良い（PWA対応）

---

## 備考

- コードがわからないメンバーが運用するため、管理画面はアプリ内の「設定」タブで完結すること
- 外部の管理画面やターミナル操作が必要な運用は避けること
- 11月の公演が終わっても次の公演に使い回せる設計にすること
